# Understanding Data Management in MoEngage

This guide will walk you through the Data Management feature in MoEngage. It helps you understand and control the data you're tracking, including events and user attributes. Think of it as your central hub for understanding what's happening with your data!

**Where to find it:** Settings > Data > Data Management

## What can you see in the Data Management dashboard?

The Data Management dashboard gives you a quick overview of your data. Here's what you'll find:

*   **Event & Attribute Usage:** A clear view of how many unique events and user attributes you're tracking compared to your allowed limit. This helps you stay within your data limits.
*   **Auto-Blocked Items:** The number of events and user attributes that have been automatically blocked because you've exceeded your limits.
*   **Data Volume:** The total amount of user and event data processed by MoEngage.
*   **Top Events:** A list of the 10 most frequent events happening in your app or website. This helps you understand what your users are doing the most.

## Navigating the Data Management Dashboard

The Data Management dashboard is organized into several tabs, each providing a different view of your data. Let's explore them:

### 1. Overview Tab: Your Data Snapshot

The Overview tab gives you a high-level view of your data usage. It's like a dashboard summarizing key metrics.

#### 1.1 Usage Section: Events and User Attributes

This section focuses on how you're using events and user attributes.

*   **Unique Events Used:** Shows the number of unique events you're tracking out of your total allowed event quota.  Think of events as actions users take in your app, like "button click" or "product purchased." Clicking this card takes you to the **Events Tab** for more details.
*   **Auto-Blocked Events:** Displays the number of events automatically blocked when you hit your unique event limit.

    **How to see the list of auto-blocked events:**

    1.  Go to the **Overview** tab.
    2.  Click the **Auto-blocked events** card.
    3.  You'll be taken to the **Event Tab** where you can see the full list.

*   **Unique User Attributes Used:** Shows the number of unique user attributes you're tracking compared to your total allowed attribute quota. User attributes are characteristics of your users, like "age," "location," or "subscription status." Clicking this card takes you to the **User Attributes Tab** for more details.
*   **Auto-Blocked User Attributes:** Displays the number of user attributes automatically blocked when you hit your unique user attribute limit.

    **How to see the list of auto-blocked user attributes:**

    1.  Go to the **Overview** tab.
    2.  Click the **Auto-blocked user attributes** card.
    3.  You'll be taken to the **User Attributes Tab** where you can see the full list.

#### 1.2 Volume Section: Data Flow

This section shows the amount of data flowing through MoEngage. Remember that changing the date range will update the charts.

*   **Event Volume (All Events):**  The total count of all events, both system and custom, being tracked.  System events are automatically tracked by MoEngage, while custom events are defined by you. Clicking this card takes you to an analytics chart where you can filter events by day.
*   **Event Volume (Custom Events):** The total count of *only* your custom events. Clicking this card takes you to an analytics chart where you can filter events by day.
*   **Web User Volume (30-Day):** The number of users who opened your web app or website in the last 30 days. A user is considered active if they open your app/site at least once a month. Clicking this card takes you to an analytics chart where you can filter users by month.
*   **Mobile User Volume (30-Day):** The number of users who opened your mobile app in the last 30 days.  A user is considered active if they open your app at least once a month. Clicking this card takes you to an analytics chart where you can filter users by month.
*   **Top Events by Volume:**  A list of the 10 events that are happening most frequently. You can filter this list to show either all events or just custom events.

### 2. Event Tab: Managing Your Events

The Event tab lists all the events you're tracking and allows you to manage them.

*   **What you'll see:** The same cards as in the Overview tab (Unique events used, Auto-blocked events).
*   **Events Not Tracked:** The number of events that haven't received any data within a chosen timeframe. If you're nearing your event limit, you can block these to make room for new events.

#### 2.1 Event Actions

You can perform the following actions on your events:

*   **Download Event List:** Click the download icon ( [CSV download icon] - replace with actual icon) to download a CSV file of your event list.
*   **Filter Events:**  Click the filter icon ( [Filter icon] - replace with actual icon) to filter events by:
    *   **Visibility:** Whether the event is visible in drop-down menus.
    *   **Category:** The pre-defined category the event belongs to.
    *   **Platform:** The platform the event is tracked on (e.g., Web, Mobile).
    *   **Last Seen:** The last time the event was tracked.
    *   **First Seen:** The first time the event was tracked.
    Click **Apply** to save your filter settings.
*   **Customize Columns:** Click the column selector icon ( [Column Selector icon] - replace with actual icon) to choose which columns you want to see in the event table.

### 3. User Attributes Tab: Managing User Data

The User Attributes tab lists all the user attributes you're tracking and allows you to manage them.

*   **What you'll see:** The same cards as in the Overview tab (Unique user attributes used, Auto-blocked user attributes).

#### 3.1 User Attribute Actions

You can perform the following actions on your user attributes:

*   **Download Attribute List:** Click the download icon ( [CSV download icon] - replace with actual icon) to download a CSV file of your user attribute list.
*   **Filter Attributes:** Click the filter icon ( [Filter icon] - replace with actual icon) to filter attributes by:
    *   **Visibility:** Whether the attribute is visible in drop-down menus.
    *   **Category:** The pre-defined category the attribute belongs to.
    *   **Platform:** The platform the attribute is tracked on (e.g., Web, Mobile).
    *   **Last Seen:** The last time the attribute was tracked.
    *   **First Seen:** The first time the attribute was tracked.
    *   **Tracked Data Type:** The type of data being tracked (e.g., String, Number, Boolean).
    *   **Allowed Data Type:** The data type that MoEngage will accept for this attribute (more on this later).
    *   **PII Masking:** Whether Personally Identifiable Information (PII) masking is enabled.
    Click **Apply** to save your filter settings.
*   **Customize Columns:** Click the column selector icon ( [Column Selector icon] - replace with actual icon) to choose which columns you want to see in the user attribute table.

## Categories: Organizing Your Data

MoEngage uses pre-defined categories to organize your events and user attributes. This helps you quickly find what you're looking for. You can expand each category to see the events or attributes within it.

**Example:** You might have categories like "Ecommerce," "Content Consumption," or "User Profile."

*   **Change Visibility:**
    *   **On:** The event/attribute will be visible in drop-down menus.

*   **Set/Edit Description:** Add context to your events/attributes. Descriptions for standard MoEngage events can't be edited.
*   **Data Privacy Configuration:**
    *   **Encrypt Attribute:** Encrypts PII (Personally Identifiable Information) attributes in the MoEngage data warehouse.  Values received *before* enabling this option will not be encrypted.

## Working with Status: Allowed and Blocked

The "Status" option lets you see whether an event, attribute, or user attribute is currently allowed or blocked.

*   **Allowed:** The attribute is being tracked and used by MoEngage.
*   **Blocked:** The attribute is *not* being tracked and is ignored by MoEngage.

**Why Block?** Over time, you might find that incorrect or test data is being tracked. Blocking these inaccurate data points helps keep your data clean.

**How to Block an Event or Attribute:**

1.  Select the event or user attribute you want to block.
2.  Click **Block**.
3.  In the confirmation dialog box, click **Update**. The status will change from "Allowed" to "Blocked."

**How to Allow a Blocked Event or Attribute:**

1.  Select the event or user attribute you want to allow.
2.  Click **Allow**.
3.  In the confirmation dialog box, click **Update**. The status will change from "Blocked" to "Allowed."

**Note:** You can use these same steps to allow events that were automatically blocked due to exceeding your data limits.

**Quick Actions:** You can also click the ellipsis (...) icon next to an event or attribute to quickly Edit, Turn visibility off, or Block.

### Blocking Due to Data Limits

If you exceed the limits for events, event attributes, or user attributes (as shown on the Overview tab), new items might be automatically blocked.

## Setting Allowed Data Types: Ensuring Data Quality

You can set the allowed data type for user attributes and event attributes. This ensures that only data of the correct type is recorded, preventing errors and improving data quality.

**How to Set Allowed Data Types:**

1.  Click the ellipsis (...) icon next to the attribute.
2.  Click **Edit attribute**.
3.  In the "Edit attribute details" window, select the desired data type from the "Allowed data type" list.
4.  Click **Save**.

### How Does This Work?

Let's say you have an attribute called "amount" and you set the allowed data type to "Numeric."  From that point on:

*   If MoEngage receives `{"amount": "145"}` (a string value), it will convert it to the number `145`. This allows you to use numeric operators in segmentation and analytics.
*   If MoEngage receives `{"amount": "sample value"}` (a string that can't be converted to a number), the value will be dropped.  MoEngage will *not* save the incorrect data.

**Data Type Conversion Table**

| Incoming Data Type | Allowed Data Type: Any | Allowed Data Type: String | Allowed Data Type: Numeric | Allowed Data Type: Boolean | Allowed Data Type: Date | Allowed Data Type: Location |
| :------------------ | :--------------------- | :----------------------- | :------------------------ | :------------------------ | :--------------------- | :-------------------------- |
| String              | Accepted as String     | Accepted as String       | Typecast to String        | Typecast to String        | Typecast to String     | Typecast to String          |
| Numeric             | Accepted as Numeric    | Typecast to Numeric      | Accepted as Numeric       | Dropped                   | Dropped                | Dropped                     |
| Boolean             | Accepted as Boolean    | Typecast to Boolean      | Dropped                   | Accepted as Boolean       | Dropped                | Dropped                     |
| Date                | Accepted as Date       | Typecast to Date         | Dropped                   | Dropped                   | Accepted as Date        | Dropped                     |
| Location            | Accepted as Location   | Typecast to Location     | Dropped                   | Dropped                   | Dropped                | Accepted as Location          |

**Important Considerations:**

*   **Default "Any" Type:** By default, attributes are set to "Any," meaning any data type will be accepted.
*   **Multiple Data Types:** If an attribute has multiple data types, your segmentation drop-down will show multiple data type options.
*   **Historical Data:**  Setting an allowed data type only affects data going forward. Older data in your database might still have different data types.  The allowed data type will be pre-selected in segmentation.

## Array Typecasting: Handling Arrays of Data

When working with arrays, all elements must be of the same data type (either all integers or all strings). If they're not, MoEngage will try to convert them.

**Example:**

If an array has elements `[1 | 2 | abc]`, and the allowed data type is "Array Integer," the final array ingested into MoEngage will be `[ 1, 2 ]`. The string "abc" couldn't be converted to an integer, so it was dropped.

## Troubleshooting

*   **Why are my events being auto-blocked?** You've likely exceeded the data tracking limits for your account.  Check the Overview tab to see your usage.  Consider blocking unused events or attributes.
*   **Why can't I see an event in the drop-down menus?**  The event's visibility might be turned off.  Go to the Event tab and check the Visibility setting for the event.
*   **Why is data not being recorded for an attribute?**  Double-check the Allowed Data Type setting for the attribute.  Make sure the data you're sending matches the allowed type.
